Activity 14: OpenStack Installation (Keystone, Glance, Nova)

1. Objectives

Create a workflow to install OpenStack using Ansible as your Infrastructure as Code

(IaC).

2. Intended Learning Outcomes

  1. Analyze the advantages and disadvantages of cloud services

  2. Evaluate different Cloud deployment and service models

  3. Create a workflow to install and configure OpenStack base services using Ansible as documentation and execution.

3. Resources

Oracle VirtualBox (Hypervisor) 1x Ubuntu VM or Centos VM

4. Tasks

  1. Create a new repository for this activity.

  2. Create a playbook that converts the steps in the following items in https://docs.openstack.org/install-guide/

    1. Keystone (Identity Service)

    2. Glance (Imaging Service)

    3. Nova (Compute Service)

    4. Create different plays in installing per server type (controller, compute etc.) and identify it as a group in the Inventory file.

    5. Add, commit and push it to your GitHub repo.

  1. Describe Keystone, Glance and Nova services


    • All of the three services mainly Keystone, Glance, and Nova Services are main components of Openstack wherein they are used in managing numerous aspects of the cloud infrastructure. such as access management, Image management, and Compute resources. Keystone provides Access management wherein user authentication and other authorization services are used. Glance is an Image service wherein Virtual machines, ImaGes and disk images are used. and final Nova Is a Compute service wherein it is responsible for provisioning virtual machines,. and provides a On demand access to computing resources such as images, and Virtual machines. Together these three contritubes to creating and managing a flexible aswell as a scalable computing environment in openstack.
